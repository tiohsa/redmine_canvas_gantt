# Epic: 表示設定を全プロジェクトで共通化

## 概要
Canvas Gantt の表示設定について、現在プロジェクトで利用中の設定を **全プロジェクト共通設定** として保存できるようにする。  
別プロジェクトを開いた際、当該プロジェクトに専用設定が存在しない場合のみ、共通設定を読み出して適用する。

本機能は **表示設定の再利用性向上** を目的とし、**プロジェクト固有のクエリやフィルタ条件は共通化対象外** とする。

---

## 背景
複数プロジェクトを横断して確認する利用者は、プロジェクトごとに毎回同じ表示設定を行う必要があり、操作負荷が高い。  
一方で、`query_id` や tracker / status / version などのプロジェクト依存条件まで横展開すると、誤適用や混乱を招く。

そのため、**Canvas 表示に関する設定のみ** を安全に全プロジェクトで共有できる仕組みを追加する。

---

## 目的
- 複数プロジェクト横断利用時の再設定コストを下げる
- 表示レイアウト・ズーム・列構成などの UI 設定を再利用可能にする
- 既存のプロジェクト単位設定と両立させる
- Zero-Migration 方針を維持する

---

## スコープ

### 対象
- Canvas Gantt の表示設定
- プロジェクト単位設定と全プロジェクト共通設定の二層管理
- 設定 UI への共通化オプション追加
- 読み込み優先順位制御
- localStorage 等の既存クライアント側保存方式での実装

### 非対象
- `query_id` の共通化
- tracker / status / version / category / assignee / custom field の具体条件の共通化
- 標準 Query の保存仕様の変更
- サーバー DB への保存
- ユーザー間共有
- 権限管理の追加

---

## 用語定義

### プロジェクト固有設定
特定 project にのみ適用される Canvas Gantt の表示設定。

### 全プロジェクト共通設定
project に依存しない表示設定のみを保持し、**プロジェクト固有設定が存在しない場合に限り** 適用される設定。

---

## 要求仕様

## 1. 設定スコープ
設定は以下の 2 階層で管理する。

1. プロジェクト固有設定
2. 全プロジェクト共通設定

---

## 2. 読み込み優先順位
Canvas Gantt 初期表示時の設定読み込み優先順位は以下とする。

1. プロジェクト固有設定
2. 全プロジェクト共通設定
3. デフォルト設定

### 補足
- 全プロジェクト共通設定が存在しても、プロジェクト固有設定がある場合は上書きしない
- 適用不能な項目があっても画面エラーにせず、該当項目のみ無視する

---

## 3. UI
設定 UI に以下のチェックボックスを追加する。

- `表示設定を全プロジェクトで共通化`

### ON 時
- 現在の設定のうち、共通化可能な表示設定のみを全プロジェクト共通設定として保存する

### OFF 時
- 以後、自動適用には利用しない
- 既存の全プロジェクト共通設定は削除せず保持してよい
- 再度 ON にした場合、保持済み設定を再利用できる構成を推奨する

---

## 4. 共通化対象
以下のような **project 非依存の表示設定** を共通化対象とする。

- ズーム倍率
- 左ペイン幅
- 表示列 ON/OFF
- 列順
- 行の表示密度
- ベースライン表示 ON/OFF
- 依存線表示 ON/OFF
- 今日線表示 ON/OFF
- 土日 / 休日表示 ON/OFF
- 折りたたみの基本表示状態
- その他 project ID や query 条件に依存しない UI 表示設定

---

## 5. 共通化対象外
以下は **共通化対象外** とする。

- `query_id`
- tracker / status / version / category / assignee の条件値
- custom field 条件
- project 固有 saved query
- project 選択条件
- project 固有のフィルタ値
- project 固有データを前提とする状態

---

## 6. 保存タイミング
保存タイミングは **明示的な保存操作時** とする。

### 理由
- 設定変更のたびに即時反映すると、意図しない global 更新が起こりやすい
- ユーザーが「この設定を共通化する」と判断して保存できる方が安全

---

## 7. 状態表示
混乱防止のため、現在どの設定が使われているかを分かるようにする。

### 表示例
- `このプロジェクト専用設定を使用中`
- `全プロジェクト共通設定を使用中`

表示形式は小さなラベル、注記、または設定ダイアログ内のステータス表示とする。

---

## データ設計方針

```ts
type ProjectScopedCanvasSettings = {
  projectId: number
  settings: CanvasDisplaySettings
  updatedAt: string
}

type GlobalCanvasSettings = {
  enabled: boolean
  settings: GlobalApplicableCanvasSettings
  updatedAt: string
}
```

### 方針
- `GlobalCanvasSettings.settings` には共通化可能項目のみ格納する
- project 固有設定と global 設定は混在させない
- 永続化先は localStorage を前提とする

---

## 想定動作

### ケース1: 共通設定の適用
1. プロジェクト A で表示設定を調整する
2. `表示設定を全プロジェクトで共通化` を ON にして保存する
3. プロジェクト B を開く
4. B に固有設定がなければ、A 由来の共通設定が適用される

### ケース2: 固有設定優先
1. 共通設定が保存済み
2. プロジェクト C には固有設定が存在する
3. C を開く
4. C の固有設定を適用し、共通設定では上書きしない

### ケース3: 共通設定が無効
1. 共通設定は保持されているが `enabled = false`
2. プロジェクト固有設定がない D を開く
3. デフォルト設定を使用する

---

## 受け入れ条件

- [ ] 設定 UI に `表示設定を全プロジェクトで共通化` が追加されている
- [ ] 共通化対象は表示設定のみに限定されている
- [ ] `query_id` やプロジェクト依存フィルタは共通化されない
- [ ] 読み込み順が `プロジェクト固有 > 全プロジェクト共通 > デフォルト` になっている
- [ ] プロジェクト固有設定が存在する場合、共通設定で上書きされない
- [ ] 共通設定が有効で、プロジェクト固有設定がない場合のみ共通設定が適用される
- [ ] OFF にしても保持済み共通設定を破壊しない
- [ ] 誤った設定適用により画面エラーにならない
- [ ] 現在どの設定が適用されているか UI 上で判別できる

---

## リスク
- 既存 localStorage 構造との互換性問題
- project 固有設定と global 設定のキー衝突
- 「現在の設定がどこから来たのか」が分からない場合の UX 混乱
- 将来、共通化対象を増やす際に project 依存項目を誤って混入させるリスク

---

## 実装方針メモ
- 既存設定オブジェクトをそのまま global に保存せず、**共通化可能項目のみ抽出して保存** する
- 読み込み時も、global 設定は **マージ対象を限定** する
- localStorage key は project スコープと global スコープで分離する
- 設定状態のラベル表示を追加する
- テストでは project 切り替え時の適用優先順位を重点確認する

---

## Issue 分解候補

### Issue 1: 設定スコープ分離
- project 固有設定と global 共通設定の保存モデルを分離する
- localStorage key 設計を整理する

### Issue 2: 共通化対象の抽出ロジック実装
- 現在設定から global 適用可能項目のみを抽出する
- project 依存項目を除外する

### Issue 3: 設定 UI への共通化オプション追加
- チェックボックス追加
- 保存時動作の接続
- ON/OFF 状態表示

### Issue 4: 読み込み優先順位の適用
- `project > global > default` の順で読み込む
- 適用不能項目の無視処理を実装する

### Issue 5: 現在適用中設定の表示
- `このプロジェクト専用設定を使用中`
- `全プロジェクト共通設定を使用中`
- 状態表示の UI 実装

### Issue 6: テスト整備
- 共通設定保存
- project 固有設定優先
- OFF 時の挙動
- 互換性確認
- 回帰テスト

---

## 非機能要件
- 既存の project 単位設定利用者の挙動を壊さないこと
- 画面初期表示で著しい遅延を生じさせないこと
- DB マイグレーション不要であること
- クライアント側のみで完結できること

---

## 判断
本 Epic は採用する。  
ただし、**「現在の設定をそのまま他プロジェクトに横流しする」機能ではなく、表示設定だけを安全に共通化する機能** として実装する。
